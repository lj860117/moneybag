## 第三步：分析 main.py 路由

### 任务目标
分析 `main.py` 中所有路由的依赖关系，为拆分做准备。

### 具体操作

1. **列出所有路由**
   - 读取 `main.py`，找出所有 `@app.get/post/put/delete` 装饰的路由
   - 统计总数，确认是否约 199 个

2. **按 URL 前缀分组**
   - 按 `/api/xxx` 的第一级路径分组（如 `/api/holdings/*`、`/api/trade/*`、`/api/portfolio/*`）
   - 列出每组包含的路由数量和 HTTP 方法

3. **识别依赖关系**
   - 每个路由函数内部调用了哪些：
     - 其他路由处理函数（直接调用）
     - 全局变量 / 模块级变量
     - 外部导入的函数（来自其他模块）
     - 数据库 / 缓存 / LLM 等 infra 调用
   - 标出**纯独立路由**（不依赖其他路由函数、只依赖标准库/infra）
   - 标出**高度耦合路由**（依赖其他路由函数或共享大量状态）

4. **建议拆分顺序**
   - 优先拆：独立、简单、调用链短的路由
   - 延后拆：耦合严重、依赖复杂的路由
   - 给出推荐的第一批拆分目标（5-10 个路由）
