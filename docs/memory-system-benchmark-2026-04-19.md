| **Claude Projects** | 1 层（CLAUDE.md） | ✅ | ❌ | ❌ | ❌ | 通用 | 几乎全部结构化层 |

| **LangGraph Memory**（LangChain 官方） | 2 层（short+long store） | ✅ | ✅ 可插 LLM | ❌ | ❌ | 通用 | 领域结构全靠自己写 |
| **Letta（前 MemGPT）** | 4 层（core / recall / archival / scratchpad） | ✅ | ✅ | ❌ | ❌ | 通用 | 我们在领域化赢；它在 hierarchy 赢 |
| **Character.ai** | 2 层（long-term + persona） | ✅ | ✅ | ❌ | ✅ 角色情感 | 对话娱乐 | profile 比我们少；家庭账号无 |

| **钱袋子（本方案）** | **9 显式 + 3 隐性** | ✅ JSON 分户 | ✅ LLM 待审 | ✅ 主账号路由 | ✅ 情绪+反讽 | **家庭理财** | ← |

### 2.2 维度详解

#### 层级丰富度
- 钱袋子 9 层 vs Mem0/Letta 3-4 层 vs ChatGPT/Claude 1 层
- **我们赢**：层级最多，每层语义最清晰
- **注意**：层多不等于好，同时要承担合成复杂度

#### 自提炼（Reflexion-style memory augmentation）
- 我们：`auto_extract_insight()` 聊天后台 LLM 自跑 → 进待审队列
- Mem0：内置 LLM extractor，直接入库
- Letta：core memory 自动压缩
- **我们的创新**：**待审队列**。不是自动入库，而是用户点头才入。这在金融场景是必须——幻觉入库比没记忆更危险
- **业界对照**：Anthropic Claude Code 也是"建议写 memory → 用户点头"模式，和我们同构

#### 多用户融合（Family CIO 机制）
- ChatGPT Teams 支持多用户账号但记忆隔离
- Mem0 有 `org_id` 但不支持主账号路由
- 钱袋子 `_route_to_admin()`：配偶产生的待审记忆/提醒都汇总到家庭主账号
- **全球独一份**。没找到第二家做夫妻共享 AI + CIO 主账号的方案

#### 情感与反讽维度
- Character.ai：有情感但服务角色扮演
- 钱袋子 `ironies`：记"用户自己打过的脸"，投资场景杀器
- **ironies 这个维度全球没第二家**。在"知错不改"的投资行为里，反讽记忆比规则记忆更有效

#### 领域本土化（农历生日）
- 所有国外方案：零支持
- 我们：`_lunar_to_solar_this_year()` 用 zhdate 库处理乐姐的农历生日
- **中国家庭场景必备**。国外方案在中国做不到的硬门槛

#### 检索效率（我们的弱项）
- Mem0 / Letta / A-MEM：向量召回，只挑相关的放 prompt
- 钱袋子：**全量 JSON 注入 prompt**。9 个文件合并成一段 summary 塞进去
- **短期 OK**：两户用户数据量 < 50KB，token 不心疼
- **长期炸**：decisions 会线性膨胀，半年后可能吃掉 20% 上下文

#### 分层压缩（hierarchy）
- Letta：core（热）/ archival（冷）/ recall（中）分层，满了自动归档
- 钱袋子：无。靠 cron 手动截断
- **我们欠一层**：archival 冷库没有

#### 图关系
- A-MEM / HippoRAG：把"茅台 → 白酒 → 消费 → 宏观"串成图，LLM 查节点不查全文
- 钱袋子：无，LLM 硬读 summary
- **学术 SOTA 差距**，工业界目前也很少用

## 4. 我们的三大核心差异化（别人抄不走）

### ★ 差异化 1：ironies（反讽记忆）
- 记录用户"自己打过的脸"
- 例："2026-03-15 雷老板说永不追高，2026-03-20 在茅台 1680 追高"
- 投资场景下比规则记忆更有效，因为规则会被情绪无视，但"自己说过的话"最能刺痛
- **全球没找到第二家做这个维度的方案**

### ★ 差异化 2：家庭 CIO 主账号路由
- 夫妻共享一个 AI
- 乐姐的待审记忆、老婆生日提醒 → 路由到雷老板
- 乐姐用得轻松，雷老板掌控全局
- **ChatGPT Teams / Mem0 org_id 都做不到**

### ★ 差异化 3：叙事式 summary（反规则化）
- LLM 会模仿 prompt 里的结构，给它【】🔒 它就返还给用户【】🔒
- 我们 V7.4.4 改成朋友介绍口吻的自然段，LLM 输出也跟着变自然
- **这是工程经验，不是论文能读到的**

## 6. 结论

### 优势（领先业内的地方）
1. **9 显式层 + 3 隐性机制**，比 Mem0/Letta 层级更丰富
2. **ironies / family_admin / 农历生日** 三项全球独有
3. **待审入库机制**和 Anthropic Claude Code 同构，哲学正确
4. **领域化深度**：家庭理财场景打磨，国外方案打不过

### 差距（落后业内的地方）
1. **无向量检索**：全量注入 prompt，token 上限受制
2. **无 hierarchy 分层**：Letta 的 core/archival 我们没有
3. **无图关系**：A-MEM 的知识图谱我们零覆盖
4. **工程成熟度低**：982 行单文件，违反内部铁律

### 一句话
> **在领域里我们是全球顶流，在基础设施上我们是业内中游。**
> 下一步应该补基础设施（向量 + 分层），而不是继续堆领域层。
