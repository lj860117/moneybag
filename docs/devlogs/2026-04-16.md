## V6 Phase 0 设计文档技术栈校准 (01:17)

**完成**：`V6-Phase0-前端整合设计.md` 升级为 v2.3（技术栈校准版）

**全面扫描项目代码后发现**：
- 后端是 **FastAPI**（非 Flask）
- 前端是**原生 JS SPA**（非 Vue）—— app.js 309KB 单文件
- 数据库是 **JSON 文件**（非 MySQL/SQLite）
- 缓存是 **内存 dict**（非 Redis）
- 推送是**企业微信**（非微信公众号）
- 部署用 **systemd**（非 supervisor）

**v2.3 核心改动**：
1. 新增章节〇：实际技术栈声明 + Phase 0 不动底层原则
2. 所有代码从 Flask/Vue/MySQL 改写为 FastAPI/原生JS/JSON
3. 前端组件 = app.js 中的 render*() 函数
4. 后端改动 = main.py 新增端点
5. 部署 = systemd restart + GitHub Actions
6. 精确到文件级的改动计划

**技术选型决策**：
- Phase 0 继续原生 JS（迁移 Vue 需 2-3 周，得不偿失）
- Phase 0 继续 JSON 文件（2 用户够用）
- Phase 0 保持 SHA256 认证（有效且在用）
- 框架迁移延后到 V8+（功能稳定后再换壳）
