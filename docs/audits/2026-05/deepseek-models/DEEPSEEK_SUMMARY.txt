MODEL SWITCHING FLOW
================================================================================

1. FRONTEND USER SELECTION
   User clicks dropdown in chat page → selects model
   ↓
   localStorage.setItem('chatModel', 'deepseek-reasoner')

2. USER SENDS MESSAGE
   sendChat() function called
   ↓
   POST /api/chat/stream with:
   {
     "message": "...",
     "model": "deepseek-reasoner",     ← User's choice
     "portfolio": {...},
     "userId": "user123"
   }

3. BACKEND PROCESSING
   backend/api/chat.py receives request
   ↓
   model = req.model or os.environ.get("LLM_MODEL", "deepseek-v4-flash")
   ↓
   Look up in AVAILABLE_MODELS:
   {
     "id": "deepseek-reasoner",
     "name": "DeepSeek R1 (深度思考)",
     "base": "https://api.deepseek.com/v1",
     "env_key": "LLM_API_KEY"
   }
   ↓
   Get API key from environment
   ↓
   Call DeepSeek API:
   POST https://api.deepseek.com/v1/chat/completions
   {
     "model": "deepseek-reasoner",
     "messages": [...],
     "stream": true
   }

4. DEEPSEEK RESPONSE STREAMING
   SSE stream with chunks:
   - reasoning_content (thinking process)
   - content (actual answer)

5. FRONTEND RENDERING
   Parse SSE chunks
   ↓
   If phase="thinking": accumulate thinkText
   ↓
   If phase="answering": accumulate fullText
   ↓
   Render with <details> block for thinking

COMMON QUESTIONS ANSWERED
================================================================================

Q: What models are available?
A: deepseek-v4-flash (V4), deepseek-v4-max (V4 Max), deepseek-reasoner (R1)

Q: Where is frontend dropdown?
A: pages/chat.js lines 1-6 (in chat page header)

Q: How do users switch models?
A: Frontend: click dropdown | WeChat: send "模型 model-id"

Q: What's special about R1?
A: Shows thinking process, 15-30s wait, collapsible in UI

Q: Which model for agent analysis?
A: deepseek-v4-max (can be overridden)

Q: How to add new model?
A: 1) Update AVAILABLE_MODELS in shared_helpers.py
   2) Add to MODEL_ROUTING in llm_gateway.py (optional)
   3) Add to MODEL_MAP in wxwork.py (for WeChat)

Q: What environment variables needed?
A: LLM_API_KEY (required), LLM_MODEL, LLM_API_BASE (optional)

Q: What about "V4 Pro"?
A: Mentioned in code but not in AVAILABLE_MODELS (likely legacy)

KNOWN ISSUES

   → Won't appear in frontend dropdown
   → Likely legacy or testing model

2. Model tier system (llm_light/llm_heavy) is separate from direct model selection

   → WeChat/internal: uses tier system
