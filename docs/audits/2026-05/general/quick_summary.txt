█ CACHE PERSISTENCE FLOW
  Startup:
    _load_cache_from_disk() → read ./data/cache/llm_cache.json
                           → restore non-expired entries
  
  Every 5 writes (in _set_cache):
    _persist_cache_to_disk() → snapshot _cache._data (with lock)
                            → write temp file + fsync + atomic rename
                            → reset _cache_dirty counter

█ MEMORY CACHE (infra/cache/memory_cache.py: MemoryCache)
  Thread-safe with threading.Lock()
  
  __init__(default_ttl=3600)
  get(key) → value or None (auto-prunes expired)
  set(key, value, ttl=0) → ttl=0 uses default, ttl<0 never expires
  delete(key), clear(), has(key), size(), keys()

█ ATOMIC WRITE PATTERN (services/persistence.py: atomic_write_json)
  1. tempfile.mkstemp() in same directory
  2. Write JSON + f.flush() + os.fsync()
  3. os.replace() for atomic rename (POSIX)
  → Prevents corruption on power loss / crash

█ KNOWN LIMITATIONS & EDGE CASES
  1. NOT thread-safe for multi-worker (race condition on _daily_count)
  2. Can't cache streaming responses (only full responses)
  3. Cache key doesn't include temperature/top_p (hardcoded at 0.7)
  4. R1 content sometimes in reasoning_content (workaround lines 184-187)
  5. Budget enforcement is soft (logs warnings, doesn't stop)
  6. Disk persist delay: max 4 calls lost if crash (every 5 writes)
