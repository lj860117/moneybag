## 🔗 模块契约（M1 W1 前：文档约束 / M1 W1 后：见 Protocol）

**上游（本模块消费谁的输出）**：
- 用户问卷输入（无上游依赖）
- `infra/store` → 文件 IO

**下游（谁消费本模块的输出）**：
- `03-rule-engine.md` → 规则引擎几乎所有规则的输入源（画像 + 资产负债表）
- `07-decision-guard.md` → 7 点清单第 1/2 条（应急金 / 四大险）直接消费本模块
- `05-scheduling.md` → A 层实时事实计算需要总资产、持仓金额来自本模块
- `04-ai-interface.md` → AI 解读层 prompt 中引用画像和负债表字段

**改动本模块前必须评估**（见 ANCHOR 改动传播表）：
- **新增/改名字段** → 同步 `03`（规则引擎读取）+ `07`（清单规则）+ 前端表单
- **过期阈值调整**（§5.1）→ 同步 `03` 的 fallback 逻辑（过期时哪些规则跳过）
- **Tier 1/2/3 调整** → 同步前端必填/选填 UI + 路线图 M2-M4
- **sub_accounts 结构** → 全局搜索 `sub_account_id` 用法

**关键不变式**：
- 过期字段**不得猜测或补算**，规则引擎必须跳过相关判断
- 子账户可**独立设目标**，但**合并计入资产负债表**

**M1 W1 后**：接口变成 `domain/protocols/family_profile.py` 里的 `FamilyProfileProtocol.load(family_id) -> FamilyProfile`，本头部瘦身为"见 FamilyProfileProtocol"。

## 二、家庭财务画像问卷（规则引擎的输入入口）

### 2.1 问题设计（10-15 题，一次性填，后续可改）

| 类别 | 问题 |
|------|------|
| 基础 | 年龄 / 家庭结构（单身/已婚/有娃/娃几岁）/ 职业稳定度 |
| 现金流 | 月收入 / 月支出 / 可投资资产总额 / 月结余 |
| 负债 | 房贷剩余 / 车贷 / 信用卡 / 其他 |
| 保障 | 重疾 / 意外 / 寿险 / 医疗（有/无 + 额度）|
| 应急 | 应急金够几个月开销 |
| 投资 | 投资年限 / 最大承受回撤（-10% / -20% / -30% / -50%）|
| 目标 | 主要目标（养老 / 买房 / 教育金 / 传承）+ 时间窗口 |

### 2.2 落地

- 模块：`domain/services/family_profile_service.py`
- 存储：`data/profiles/{family_id}.json`（通过 `infra/store`）
- 前端：一个问卷页，可随时修改

### 2.3 数据结构（预留家庭成员 / 子账户维度）

现在可以只做主账户，但结构先留好，否则后面重构代价大：

```json
{
  "family_id": "LeiJiang-family",
  "members": [
    {"member_id": "self", "role": "主申请人", "age": 38, "income": 25000, "is_decision_maker": true},
    {"member_id": "spouse", "role": "配偶", "age": 36, "income": 18000},
    {"member_id": "child1", "role": "子女", "age": 6}
  ],
  "sub_accounts": [
    {"account_id": "main", "purpose": "家庭主账户", "target_allocation": {...}},
    {"account_id": "education", "purpose": "教育金（目标: 2040 年 100w）", "horizon_years": 15},
    {"account_id": "pension_parents", "purpose": "父母养老", "is_independent": true}
  ]
}
```

**原则**：家庭合并看资产负债表，但**子账户可独立设目标和配比**。教育金 15 年周期 vs 养老 30 年周期，不该混在一起算偏离度。

## 三、家庭资产负债表（全景视角）

### 3.1 为什么要做

当前系统只盯"股票+基金持仓"，但家庭资管最该看**整张资产负债表**。

### 3.2 字段全集

```
资产：现金 / 理财 / 股票 / 基金 / 黄金 / 房产自住 / 房产投资 / 保单现金价值
负债：房贷 / 车贷 / 消费贷 / 信用卡
保障：重疾额度 / 寿险额度 / 医疗险 / 意外险
现金流：月收入 / 月支出 / 月结余 / 应急金月数
```

### 3.3 AI 价值点

从整张表发现"真问题"而不是"假问题"。典型产出：
- "你股票仓位纠结 3% 偏差，但家里没有重疾险，本末倒置"
- "你应急金只够 1.5 个月，应先把应急金补到 6 个月再加仓"
- "你自住房占总资产 70%，金融资产里该更保守（已超配不动产）"

## 四、最小可用版本（MVP）—— 对抗维护成本黑洞

**现实**：全量资产负债表的最大落地风险是**用户不会长期维护**。股票基金能自动同步，但房产估值 / 保险保单 / 线下理财 / 现金流 **几乎必须手动**。用户坚持不了 3 个月，最后只有股票数据是实时的、其他全是半年前的，整个"家庭管家"逻辑崩塌。

**设计原则**：**宁可字段少而新，不要字段全而旧**。

### 4.1 Tier 1：必填（4 类，MVP 阶段只做这些）

| 字段 | 维护方式 | 摩擦 |
|------|---------|------|
| 现金 / 理财 / 余额宝等 | 总额一个数，月度更新 | 极低 |
| 股票 / 基金持仓 | 手动录入 or 导入券商流水 | 中（已有模块）|
| 房贷余额 | 月度从银行 App 抄一个数 | 低 |
| 重疾 / 寿险 / 医疗 / 意外 额度 | **一次录入多年不变**（保单期内）| 极低 |

### 4.2 Tier 2：选填（填了好，不填有 fallback）

| 字段 | 建议频率 | 不填时 fallback |
|------|---------|----------------|
| 自住房估值 | 年度（或大涨大跌时）| 用购入价 × 地区年化 3% 粗估 |
| 投资房 / 商铺 | 年度 | 同上 |
| 车贷 / 消费贷 / 信用卡 | 月度 | 不计入 |
| 月收入 / 月支出 | 月度 | 不算应急金月数 |
| 保单现金价值 | 年度 | 不计入流动资产 |

### 4.3 Tier 3：不做

| 字段 | 为什么不做 |
|------|----------|
| 具体个股成本明细（已在股票模块）| 重复 |
| 每笔流水 | 个人用户不会记，费心累 |
| 遗产 / 信托 / 税务 | 远超普通家庭使用场景 |

## 五、Stale（过期）标识与 Graceful Fallback

### 5.1 字段过期阈值

写在 `domain/rule_engine/defaults.py`：

| 字段类 | 过期阈值 | 过期后行为 |
|--------|---------|----------|
| 股票基金（自动）| 1 天 | 白天展示昨日 + 角标 |
| 现金/理财 | 90 天 | 灰色展示 + "更新一下" 提醒 |
| 房贷余额 | 60 天 | 灰色展示 + 提醒 |
| 保险额度 | 365 天 | 灰色展示 + 提醒 |
| 房产估值 | 365 天 | 自动衰减 + 灰色 |
| 收支 | 90 天 | 规则引擎**跳过应急金相关判断** |

### 5.2 规则引擎对过期数据的处理原则

1. **不猜、不补**：没数据或数据过期 → 对应规则不触发，而不是用旧数据算出错误结论
2. **在 UI 上明确标注**："应急金判断已跳过（收支数据过期）"
3. **引导用户更新**：过期字段上"一键更新"按钮，点击直接定位到问卷对应题

## 六、最低摩擦的填写体验（产品设计）

决定用户能不能坚持的不是字段设计，是**填写摩擦**。

| 场景 | 旧（高摩擦）| 新（低摩擦）|
|------|-----------|-----------|
| 房产估值 | 要求精确到元 | **"估个大概，100 万还是 200 万？"** + 滑块 |
| 家庭保障 | 要求填保单号/到期日 | **"有没有 + 大概保多少"** 两个字段 |
| 月度更新 | 手动打开 App 填 | **月初推一条"花 1 分钟更新下家底"**，直接跳到最需要更新的 Tier 1 字段 |
| 数据"真实性" | 追求精确 | 接受 ±10% 误差，大方向对就行 |

**核心产品信条**：**月度 1 分钟，好过一次 30 分钟再也不来**。

## 七、分期上线

| 阶段 | 版本 | 内容 |
|------|------|------|
| M2 | MVP | Tier 1 四类必填 + 过期标识 + 每月提醒 |
| M3 | 扩展 | Tier 2 选填字段 + 一键更新入口 |
| M4+ | 精细化 | 收支流水的轻度记录（可选）、保单到期提醒 |

**M2 只做 MVP**，不要在入口层卷复杂度。
