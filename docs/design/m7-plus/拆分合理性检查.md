## 二、耦合遗漏分析（已修复）

### ~~遗漏 1~~ ✅ 已修复：凌晨工厂统一接入接口

**原问题**：Batch 1 和 Batch 7 各自描述凌晨工厂接入方式，但没有统一的接口契约。

**修复措施**：
- 新建 `00-interface-contracts.md` §一，定义 `NightWorkerStep` Protocol、`PipelineContext`、`StepResult`、`StepStatus` 完整签名
- 定义各批次的实现映射（step_name、target_stage）
- 定义注册方式（步骤列表扩展 vs 硬编码降级）
- 定义错误隔离保证（`run_stage_steps` 执行器）
- Batch 1（01-batch）和 Batch 7（07-batch）末尾均添加 `⚙️ 全局契约引用` 指向 00-interface-contracts.md §一

### ~~遗漏 2~~ ✅ 已修复：defaults.py 新增 dataclass 跨批次约定

**原问题**：多个批次并行开发可能产生命名冲突。

**修复措施**：
- 在 `00-interface-contracts.md` §二 定义完整规范：
  - 命名格式：`{功能域}Defaults`（PascalCase）
  - 注释标记：`# --- Batch N: 简述 ---`
  - docstring：必须注明 `M7+ Batch N: 用途`
  - 排列顺序：按批次编号
  - frozen=True 要求
  - 合并策略：不同批次 dataclass 名称不冲突
- 列出所有批次的 dataclass 完整定义（含字段名、默认值、注释）
- 所有 8 个批次文件末尾均添加 `⚙️ 全局契约引用` 指向 00-interface-contracts.md §二
