## 一、核心认知转变

### 1.1 AI 的底层宿命

AI（大语言模型）是**文本概率 Transformer**，天生：
- ❌ 不会数学统计、方差、波动率、概率分布
- ❌ 不理解金融时序的混沌非平稳特性
- ❌ 从噪音中找规律 → 严重过拟合 → 回测神仙、实盘亏钱

**结论**：AI 做时序定价、择时选股、盘口预测 → 极弱，不可依赖。

### 1.2 AI 真正能做的 4 件事

| 能力 | 落地方式 | moneybag 对应 |
|------|---------|--------------|
| 写代码 | 生成回测框架、策略代码、数据清洗脚本 | 开发时辅助工具 |
| 逻辑提炼 | 人的交易想法 → 数学公式/规则 | 策略想法 → 代码实现 |
| 风控建模 | 仓位、分散、对冲、回撤控制规则 | 规则引擎层 |
| 复盘归因 | 历史决策 + 后续走势 → "为什么错" | 复盘报告生成 |

### 1.3 AI 绝对不能做的 3 件事

| 禁区 | 现状踩线情况 | 后果 |
|------|------------|------|
| 直接喂全量 K 线让 AI 分析涨跌 | `ai_predictor.py` MLP+GBM 预测 | 过拟合、风格失效 |
| 海量时序数据塞进上下文选股 | `decision_maker.py` 把数据塞给 R1 | 看图说话、历史拟合 |
| 复杂多因子让 AI 自动拟合策略 | `ai_predictor.py` 40+特征集成 | Token 稀释、逻辑混乱 |

### 1.4 AI 的新边界（家庭资管场景）

**核心原则**：AI 可对**框架**做判断，不能对**证券**做预测。

| 能做（金融常识的个性化应用）| 不能做（对证券做预测）|
|---|---|
| 40 岁有孩家庭股债金该怎么摆 | 茅台明天涨不涨 |
| 为什么保险比股票更优先 | 某只基金未来一年收益多少 |
| 应急金该 3 个月还是 6 个月 | 现在该不该加仓创业板 |
| 这次加仓会不会打破分散原则 | 下周市场会涨还是会跌 |

成熟金融知识（现代组合理论、生命周期投资、家庭财务金字塔）**不是预测，是常识的个性化投递**。

## 二、定位重塑：家庭资产管理工作台

### 2.1 从 → 到

| 旧定位 | 新定位 |
|--------|--------|
| AI 量化交易系统 | AI 辅助投资信息工作台 |
| AI 帮你选股赚钱 | AI 帮你理清家底，你自己决定怎么配 |
| 预测涨跌、精准推票 | 资产配置体检、方向筛选、候选池展示 |

### 2.2 核心设计原则

1. **不做择时**：固定周期定投/再平衡，不预测明天涨跌
2. **不给仓位**：只给方向和大类比例，具体买多少人定
3. **极简 AI**：AI 只翻译已有数据和规则，不创造结论
4. **规则兜底**：所有计算用 Python，所有判断用规则引擎

## 三、未来架构（四层）

```
┌─────────────────────────────────────────────┐
│  第四层：人做决策                              │
│  看报告 → 自己决定买哪个、买多少、什么时候调仓    │
└─────────────────────────────────────────────┘
                      ↑
┌─────────────────────────────────────────────┐
│  第三层：AI 解读层（LLM，只说话不决策）          │
│  - 资产配置体检报告翻译                        │
│  - 市场环境总结（白话版）                       │
│  - 持仓健康度提醒                              │
│  - 风险清单                                   │
│  - 候选池点评（为什么值得关注）                  │
│  - 复盘归因报告                               │
│  ❌ 不输出：action/position_pct/涨跌预测        │
└─────────────────────────────────────────────┘
                      ↑
┌─────────────────────────────────────────────┐
│  第二层：规则引擎（程序执行，无AI）              │
│  - 资产配置配比计算（目标 vs 实际偏差）          │
│  - 超配/欠配提醒                              │
│  - 行业/方向筛选（regime + 资金 + 估值）        │
│  - 5 维评分排序                               │
│  - 风控规则（集中度、止损、回撤）                │
│  - 再平衡提醒（固定周期）                       │
└─────────────────────────────────────────────┘
                      ↑
┌─────────────────────────────────────────────┐
│  第一层：数据采集与预处理（Python，无AI）         │
│  - Tushare / AKShare 行情/资金/研报采集         │
│  - 数据清洗、标准化、缓存                       │
│  - 技术指标计算（RSI/MACD/波动率等）             │
│  - 特征工程（给人和规则引擎用，不给AI预测）       │
└─────────────────────────────────────────────┘
```

**这是业务架构。代码架构（api / use_cases / domain / infra）见 `12-framework-refactor.md`。**

## 四、"有意义建议"的边界

### 4.1 AI 能给出的（高质量）

| 类型 | 示例 |
|------|------|
| 配置体检 | "你的股票仓位超配 13%，建议再平衡" |
| 风险提醒 | "持仓集中度 60%，分散不足" |
| 方向筛选 | "银行板块估值 10% 分位，红利方向值得关注" |
| 候选点评 | "该方向 XX 基金评分最高，费率低、规模大" |
| 复盘归因 | "上次买入时 RSI 超买，未设止损，回撤 -20%" |

### 4.2 AI 给不出的（别指望）

| 类型 | 为什么 |
|------|--------|
| "明天这只票涨不涨" | AI 不懂时序 |
| "现在加仓 3 成" | AI 不知道你的现金流和心理承受力 |
| "这只基金未来收益 15%" | 过拟合，骗你的 |

### 4.3 核心认知

> **AI 不能帮你"选对"，但能帮你"别犯蠢"。**
>
> 对家庭资产管理，"别犯蠢"（仓位不超配、不集中度过高、有纪律再平衡）比"选对"（买到暴涨票）重要 10 倍。

## 五、一句话总结

> **AI 做你的财务管家（整理信息、提醒偏差、解释方向），你才是 CFO（拍板买卖）。**
>
> 数据采集用爬虫/API，指标计算用 Python，综合判断用规则引擎，AI 只负责把结果翻译成人话。
>
> DeepSeek 不在白天柜台算命，在凌晨仓库批量翻译——这才叫用对模型。
>
> 不懂金融不是缺陷，是起点。AI 的职责不是替你永远决定，是让你一年后自己也能看懂报告。
