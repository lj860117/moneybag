## 🎯 快速诊断指南

### 场景 1：用户说"收不到晨报"

**第一步**：检查 night_worker 日志
```bash
tail -100 ~/WorkBuddy/moneybag-for-claudecode/data/night_worker/$(date +%Y%m%d).log
```

**关键词查找**：
- "❌ 保存简报失败" → 参考 **03_SOLUTIONS_AND_FIXES.md 修复 #1**
- "无简报文件，跳过" → 参考 **03_SOLUTIONS_AND_FIXES.md 修复 #2**
- "Phase 1 失败" → 参考 **02_ROOT_CAUSE_ANALYSIS.md 层级性质**

---

### 场景 2：用户收到"暂无分析"的晨报

**第一步**：检查数据源
```bash
grep "数据源异常" ~/WorkBuddy/moneybag-for-claudecode/data/night_worker/$(date +%Y%m%d).log
grep "Regime" ~/WorkBuddy/moneybag-for-claudecode/data/night_worker/$(date +%Y%m%d).log
```

**原因**：
- 数据源故障导致 Phase1-3 返回空值
- 系统继续执行（"graceful degradation"）但用户看不到

**解决**：参考 **01_FAILURE_TRACE_ANALYSIS.md** 的诊断点 2-4

---

### 场景 3：晨报延迟超过 50 秒

**第一步**：检查模块超时
```bash
grep "timeout 5s" ~/WorkBuddy/moneybag-for-claudecode/data/night_worker/$(date +%Y%m%d).log | wc -l
```

**如果数值 > 3**：模块普遍超时
- 参考 **01_FAILURE_TRACE_ANALYSIS.md 失败点 #4**
- 参考 **03_SOLUTIONS_AND_FIXES.md 修复 #5**

---

### 场景 4：磁盘错误导致简报丢失

**症状**：
- 凌晨 06:00 日志显示脚本异常退出

**根本原因**：

**解决**：参考 **03_SOLUTIONS_AND_FIXES.md 修复 #1**

## 📞 常见问题

**Q: 为什么系统不直接抛异常而是"无声失败"？**
A: 这是有意设计的"优雅降级"（graceful degradation），但对用户不透明。
参考：**02_ROOT_CAUSE_ANALYSIS.md - 根本原因 #1**

**Q: 为什么用户中午收到的仍是早晨的坏报告？**
A: 缓存没有过期检查。同一天内的请求会命中缓存，即使数据源已恢复。
参考：**02_ROOT_CAUSE_ANALYSIS.md - 根本原因 #2**

**Q: 为什么用户有时一点反应都没有（推送失败）？**
A: 简报文件写入或推送过程中的异常没有被捕获和告警。
参考：**02_ROOT_CAUSE_ANALYSIS.md - 根本原因 #3**

**Q: 修复这些需要多久？**
A: P1+P2 修复 2 小时内完成。完整修复 6 小时。
参考：**03_SOLUTIONS_AND_FIXES.md - 完整修复清单**
